# FiltersView

Route `/filters`. Quick views plus the AI's smart tags.

## Sections

- **My filters** — Priority / Deadline / By smart tag. Each is a toggle; the
  count on the right is live.
- **Labels** — the fixed 5-tag taxonomy as pills with live counts.
- **Results** — appears only when a filter or label is active.

Filters and labels are mutually exclusive; selecting one clears the other.

## Counts

All computed from open (non-done) tasks:

| Filter | Count |
|---|---|
| Priority | tasks with `priority === "high"` |
| Deadline | tasks with a `deadline` |
| By smart tag | number of **distinct** tags in use |

## Gotchas

- The tag taxonomy is fixed at 5 (`TAGS` in `src/lib/types.ts`) and the model
  is constrained to it by the output schema. Labels are "auto-generated by
  Cerno" in the sense that Cerno *assigns* them — it cannot invent new ones.
- Label colours live in `src/lib/labels.ts`, never in the database.
